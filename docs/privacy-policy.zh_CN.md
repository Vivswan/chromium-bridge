# 隐私政策：Chromium Bridge

> 本文是 [privacy-policy.md](./privacy-policy.md) 的简体中文翻译。以英文版
> 为准；繁體中文版见 [privacy-policy.zh_TW.md](./privacy-policy.zh_TW.md)。

_最后更新：2026-07-17_

Chromium Bridge 是一个开源浏览器扩展，通过一个运行在你自己电脑上的原生消
息主机，把 MCP 客户端（如 Claude Code、Claude Desktop 或 Codex）连接到你本
地的 Chromium 浏览器。本政策说明扩展会访问什么，以及它对这些数据做什么、
不做什么。

## 摘要

**Chromium Bridge 不收集、不传输、不出售任何个人数据。** 它没有分析统计、
没有遥测、没有远程服务器。扩展做的一切都发生在你自己的机器上：站点只有在
你批准之后才能被操作，每个高风险操作都会请求你的确认。

## 扩展能访问什么

为了让经你批准的 AI 代理操作你已登录的页面，扩展可以（**仅限你明确批准的
站点**；或在你显式开启 `allowAllSites` 后，所有站点）：

- 读取当前页面的内容（DOM、文本、表单字段）。
- 读取活动站点的 Cookie，包括 `httpOnly` Cookie。
- 读取 Web 存储（`localStorage` / `sessionStorage`）。
- 在页面中执行 JavaScript。
- 把本地文件附加到页面的文件输入框（默认关闭；每次使用都显示确切路径供确
  认）。

与站点批准无关，它还可以读取打开的标签页列表（标题和 URL），以及打开、切
换或关闭标签页；关闭标签页会请求确认。

含凭据的值（Cookie 和 Web 存储）是**只读的**（扩展在设计上没有写入或修改
Cookie 或存储的 API），并在返回前**脱敏**（JWT、长十六进制串、长数字串会
被抹除）。

## 数据如何使用、去向哪里

- 所有通信都留在**你自己的电脑上**。扩展经 Chrome 的原生消息通道与本地原
  生消息主机通信；该主机经一条私有的、经过身份认证的本地套接字与你的 MCP
  客户端通信。
- **扩展不向其作者或任何自有服务器发送任何数据。** 它自身不发起任何对外网
  络请求。代理读取的内容会在同一台机器上、应你的请求返回给你配置的 MCP 客
  户端；该客户端接下来如何处理这些数据（例如发送给它使用的 AI 服务），由
  该客户端自己的隐私政策约束，而不在本政策范围内。

## 同意与控制

- **逐站点批准。** 在提示中批准某个源之前，该站点的页面无法被读取或操作。
- **逐操作确认。** 高风险操作（表单提交、按键、关闭标签页、文件上传，以及
  每一次 JavaScript 执行）都会在一个网页无法看见、无法交互的扩展自有窗口
  中请求确认。在已完成 Touch ID 注册的 Mac 上，最高风险的操作需要
  Touch ID 批准。这些确认默认开启；每一项都是由你控制的设置。
- **紧急停止开关。** 你可以随时从扩展的选项页、桌面应用或命令行停止桥的一
  切活动；解除需要你本人在场的明确批准。

## 扩展在本地存储什么

扩展只在你的设备上、浏览器的本地扩展存储中保存少量配置：

- 你批准的站点列表（允许列表）。
- 你的扩展设置/偏好。
- 如果你完成了注册：用于验证你自己电脑安全硬件的公钥指纹（绝不是私钥：私
  钥留在 Secure Enclave 里）。
- 一份有界的、本地的近期安全决策日志（确认、吊销），可在选项页查看。

这些数据绝不离开你的设备，并在你卸载扩展时被移除。

## 远程代码

扩展**不**加载或执行远程托管的代码。可能在页面中被执行的 JavaScript 是你
（或在你指示下的 MCP 客户端）在本地提供的代码，绝不会从远程来源获取。

## 数据共享与出售

Chromium Bridge **不**向任何人出售或共享你的数据。唯一会收到数据的一方，
是你自己配置并接入这座桥的 MCP 客户端。
