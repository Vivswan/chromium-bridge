Chromium Bridge 设置

所有更改即时保存。安全类开关请谨慎调整。

安全

要求主机配对(Secure Enclave 注册)
开启时,只有完成一次配对仪式并固定了主机公钥后,桥接请求才会被执行(ADR-0021)。配对能阻止静默配对——恶意的 claude mcp add 或任何未经你批准的主机都无法驱动浏览器;配合下方「立即验证」也能发现主机被替换。它不能阻止已配对之后同用户进程的静默替换(那需要每操作 Touch ID,是另一个未实现的可选层级)。仅 macOS 有 Secure Enclave:其他平台上配对不可用,此项不会阻断桥接——桥接以基础认证运行(Linux:UDS + peer-UID + HMAC + 二进制签名校验;Windows:仅 HMAC),主机被替换的残余风险见威胁模型。判定用的是浏览器自身的平台信息,不采信主机的自述,因此 macOS 上的主机谎称「不支持」不能绕过配对。
⚠ 关闭后,任何能写 MCP 配置的本地进程都可能在你不知情时建立并使用桥接。
允许 page_eval(执行任意 JavaScript)
关闭后,AI 调用 page_eval 会直接被拒绝,不会在页面执行任何代码。这是最危险的工具——即使有确认通道,关闭它可彻底消除静默代码执行的风险(ADR-0008)。
脱敏 page_eval 返回值
对 eval 结果做正则遮罩:JWT、长 hex、长数字、疑似 token/密钥的字符串(ADR-0008)。
⚠ 关闭后,token / Cookie / 密钥可能原样泄露给 AI。
高危点击二次确认(提交按钮 / 链接跳转)
点击表单提交按钮或链接时弹出确认 Toast,需用户批准。这是防止 AI 静默下单/转账/跳转的关键防线(ADR-0006)。
⚠ 关闭后,AI 点击提交/链接将不再确认,请仅在你完全信任 AI 时关闭。
page_eval 二次确认(执行任意 JS)
每次 page_eval 弹出确认 Toast 并展示完整代码,需用户批准(ADR-0008)。page_eval 能执行任意 JavaScript,是风险最高的操作。
⚠ 关闭后,AI 将无提示直接执行任意 JS,请仅在完全信任 AI 且需要全自动时关闭。
tab_close 二次确认(关闭标签页)
关闭标签页前在页面内弹出「Close tab?」确认,需用户批准。
⚠ 关闭后,AI 将无提示直接关闭标签页。
精确 snapshot 前提示
调用 page_snapshot_precise(走 chrome.debugger)前在页面弹出信息提示。Chrome 顶部会短暂显示「调试中」横幅(ADR-0009)。

主机配对(Secure Enclave)

流程:先在终端运行 chromium-bridge pair(会弹 Touch ID,并打印公钥指纹),然后点击下方「开始配对」,再次通过 Touch ID 后,这里会显示主机的指纹。逐字符对比两处指纹,一致才批准;批准后公钥被固定。请注意默认行为的边界:主机身份只在配对时(有人在场)和你手动「立即验证」时被验证,重连不会自动重新验证——每次验证都要弹 Touch ID,而 MV3 每几分钟就重连一次,逐次验证不可用(ADR-0021)。因此配对之后主机若被同用户进程替换,默认不会被自动发现,直到你手动验证、开启下方的定期重新验证,或未来的每操作确认层级。

大于 0 时,重连若距上次成功验证超过该间隔,会自动对固定公钥发起一次验证(弹 Touch ID;例如 3600000 = 每小时至多一次)。这能把「主机被替换却未被发现」的窗口限制在该间隔内;验证失败会立即阻断桥接。未通过的提示(比如你随手取消)不改变已配对状态,只有密码学上的不匹配才会阻断。

标签页

把 AI 打开的标签页归入「Chromium Bridge」分组
通过 tab_open 打开的标签页会自动放进一个命名的标签分组,和你自己的标签页视觉隔离,可整组折叠/关闭。默认开启(ADR-0018)。

执行模式

全部页面操作走 CDP(chrome.debugger)
开启后所有页面操作改用 CDP(chrome.debugger),不再注入内容脚本;可在严格 CSP 站点(如 Bing)执行 page_eval。默认关闭(ADR-0017)。
⚠ 开启后 Chrome 顶部会持续显示「正在调试此浏览器」横幅(调试器全程附加),且访问面更大。仅在需要绕过 CSP 或统一深控时开启。
允许 page_upload(上传本地文件)
开启后 AI 可用 page_upload 把本机文件附加到页面的文件输入框并上传。默认关闭。这是本地文件外泄通道:被劫持的模型可借此上传你的私有文件。每次调用仍会在页面弹出确认并展示完整文件路径。
⚠ 开启后 AI 能读取并上传本机文件(逐次确认路径)。仅在你确实需要让 AI 上传指定文件时开启。
允许 page_handle_dialog(响应弹窗)
开启后 AI 可用 page_handle_dialog 接受/取消页面的 alert / confirm / prompt 弹窗(走 chrome.debugger)。默认关闭。弹窗会阻塞页面,无法在页面内二次确认,因此这个开关本身就是唯一的门禁。
⚠ 接受弹窗可能替你确认危险操作,且无法在页面内二次确认。仅在你信任 AI 且确有需要时开启。

工具启用

取消勾选的工具,AI 调用时会收到「tool disabled in settings」错误。注意:AI 仍能看到该工具存在,但无法执行。

允许的站点(白名单)

允许所有站点(跳过逐站点审批)
开启后,AI 可直接操作任意 http(s) 站点,不再为每个新站点弹窗授权。开启时会请求「<所有网址>」主机权限。适合信任 AI 全自动浏览的场景;否则建议保持关闭,用下面的逐站点白名单。
⚠ 开启后,任意站点(含银行、邮箱、内网)都无需授权即可被操作,请谨慎。
格式:https://域名/*。手动添加后,首次访问该站点时仍需批准 Chrome 的主机权限弹窗。
